# 指標とディメンション {#concept_rfj_wpt_52b}

>[!NOTE]
>
>配信データは、Adobe Campaign Standardを使用している場合にのみ入力されます。

動的レポートで使用できるすべてのコンポーネントとその定義は、こちらで確認できます。

次の表に、旅行レポートで使用されるディメンションとその定義を示します。

ディメンションと指標の互換性の詳細については、このページを参照 [してください](../assets/do-not-localize/dynamic_report_compatibility_journey.pdf)。

## ジャーニーディメンション {#MBE_table_wk4_bnj_w2b}

次の表に、遍歴レポートで使用されるディメンション、その定義および式のリストを示します。

| サイズ | 定義 |
|--- |--- |
| **アクション** | ジャーニーで使用されるすべ&#x200B;**てのアクション(アクション名**— アクションラベル)のリスト（例：プッシュ — チェックアウト確認、電子メール — 報酬の忠誠度）。 |
| **データソース** | エクスペリエンスプラットフォ&#x200B;**ーム、予約システムなど**、旅行でデータを拡張するために使用するデータソース（データソース名）のリストです。 |
| **[!UICONTROL Event]** | ジャーニーで使用され&#x200B;**るすべてのイベント(イベント名**— イベントラベル)のリスト（例：Geometrixxイベント — Geometrixxチェックアウト）。 |
| **フィールドグループ** | ジャーニーでデータを&#x200B;**豊富にするために使用するフィールドグループ**（フィールドグループ名）のリストです（例：プロファイルフィールドグループ、Geometrixx予約システム）。 |
| **旅** | テストモードとライ&#x200B;**ブでの各遍歴(ジャーニー名**)のリスト（例：買い物かごの放棄、ホテルの予約通知）。 |
| **ジャーニーバージョン** | 旅行の公開バージョン(旅行名+バ&#x200B;**ージョン番号**)のリスト（例：買い物かごの放棄v1、ホテルの予約通知v2）。 |
| **オーケストレーション** | ジャーニーで定義および使用され&#x200B;**るすべてのオーケストレーションアクティビティ(Condition、End**、Wait)のリストです。 |

## 配信分析コード {#delivery-dimensions}

次の表に、遍歴レポートで使用される配信ディメンション、その定義および式を示します。

| ディメンション | 定義 |
|--- |--- |
| **ブラウザー** | メッセージが開かれた、またはクリックされたブラウザー。 |
| **配信名** | 配信のラベルとID。 |
| **デバイス** | 電子メール/SMS/プッシュ通知が開かれ、表示、またはクリックされたデバイス。 |
| **メッセージタイプ** | 電子メール、SMS、プッシュ通知、アプリ内通知など、配信に使用するチャネル。 |
| **モバイルアプリ名** | モバイルアプリケーションの名前 |
| **プラットフォーム** | メッセージの開封、表示、クリック元のデバイスのプラットフォーム。 |
| **[!UICONTROL Push platform]** | プッシュ通知が開かれたデバイスのプラットフォーム（iOS、Androidなど）。 |
| **受信者ドメイン** | 電子メールを開くために使用するドメイン。 |
| **トラッキング URL** | ユーザーがメッセージからクリックしたURL。 |
| **追跡URLカテゴリ** | 追跡URLに割り当てられたカテゴリ。 |
| **追跡URLラベル** | URLに付与されるラベル（ミラーページなど）は、お問い合わせいただくか、開いてください。 |
| **バリアント** | A/Bテストの場合の電子メールのバリアントです。 |


## 遍歴指標 {#MBE_p_p22_c4j_w2b}

次の表に、旅行レポートで使用される指標、その定義および数式のリストを示します。

| 指標 | 定義 |
|--- |---|
| **完了** | 通常の旅を終えた人の総数。 |
| **完了率** | 通常は旅を終えた人の総数で、旅に出た人の総数と比較した。 |
| **現在** | 現在、旅行中の人数の合計です。例えば、入口にした人数から退出した人数、エラーおよびタイムアウトを引いた人数です。 |
| **現在のレート** | 現在、旅に出た人の総数と、旅に出た人の総数。 |
| **入力済み** | 遍歴内の個々のエントリを開始するために発生したイベントの総数。 |
| **エラー** | 旅行中に発生したが、その旅行の成功を妨げなかったエラーの総数です。 |
| **アクションのエラー** | アクションに対して発生したエラーの合計数です。 |
| **濃縮誤差** | データソース/フィールドグループの呼び出し時にデータリッチメントに対して発生したエラーの合計数です。 |
| **イベントのエラー** | イベントに対して発生したエラーの合計数。 |
| **エラー率** | 旅行中に発生したエラーの総数で、その旅行中の合計数と比較したもの。 |
| **実行されたアクション** | 1回の遍歴に対して実行されたアクションの総数です。 |
| **実施済利益** | データソースを呼び出して特定のフィールドグループを取得した結果実行されたリッチメントの合計数です。 |
| **実行されたイベント** | 1回の遍歴に対して実行されたアクションの総数です。 |
| **実行されたオーケストレーション** | 1回の遍歴に対して実行されたオーケストレーションオブジェクト（終了、待機、条件）の合計数です。 |
| **失敗** | 正常に実行されなかったジャーニーの合計数です。 |
| **失敗率** | 正常に実行されなかったジャーニーの合計数（走行ジャーニーの数と比較）。 |

## 配信指標 {#delivery-metrics}

次の表に、ジャーニーレポートで使用される指標、その定義および数式のリストを示します。

| 指標 | 定義 |
|--- |--- |
| **ブラックリスト** | 電子メールをスパムまたは迷惑メールとして宣言した受信者の数です。 |
| **ブラックリスト記載率** | ブラックリストに記載されたメッセージの合計数で、送信されたメッセージと比較したもの。 |
| **バウンス+エラー** | 送信されたメッセージの合計数に関して、配信および自動返信処理中に累積されたエラーの合計。 |
| **直帰率+エラー率** | 送信されたメッセージと比較してバウンスしたメッセージの合計数です。 |
| **クリック** | 配信でコンテンツがクリックされた回数。 |
| **クリックスルー率** | 配信されたメッセージの数と比較した、配信のクリック数の合計です。 |
| **配信済み** | 送信されたメッセージの合計数に関連して正常に送信されたメッセージの数。 |
| **配送率** | 送信されたメッセージと比較して正常に配信されたメッセージの合計数です。 |
| **エラー** | 旅行中に発生したが、その旅行の成功を妨げなかったエラーの総数です。 |
| **ハードバウンス** | 誤った電子メールアドレスなどの恒久的なエラーの合計数。 |
| **直帰率** | 送信されたメッセージと比較して、永続的なエラーが原因で失敗した配信の合計数です。 |
| **ミラーページ** | ミラーページのリンクをクリックした受信者の数。 |
| **ミラーページレート** | 配信されたメッセージの合計と比較した、ミラーページリンクのクリック数の合計。 |
| **開封数** | 配信でメッセージが開かれた回数。 |
| **オープン率** | 配信されたメッセージの数と比較した、開かれたメッセージの合計数です。 |
| **強制隔離** | アドレスの検疫につながった、バウンスしたメッセージの数です。 |
| **検疫率** | 送信されたメッセージと比較した検疫の合計数です。 |
| **却下** | SMTPサーバーによってスパムとして分類されたメッセージの数です。 |
| **却下率** | 拒否とマークされたメッセージの合計数で、送信されたメッセージと比較します。 |
| **処理済み/送信済み** | 配信の送信の合計数。 |
| **ソフトバウンス** | 完全な受信トレイなどの一時エラーの合計数です。 |
| **ソフトバウンス率** | 一時的な理由によって失敗した、送信されたメッセージと比較した配信の合計数です。 |
| **個別クリック数** | 配信内のコンテンツをクリックした受信者の数。 |
| **個別オープン** | 配信を開いた受信者の数。 |
| **購読解除済み** | 購読解除リンクのクリック数。 |
| **登録解除率** | 配信されたメッセージと比較した、受信者による購読解除の合計数です。 |
